关于Mysql的truncate操作

发表于 2019-08-14 分类于 Mysql

在公司项目的旧代码中看到有mysql的truncate操作。
逻辑就是truncate操作之后，执行任务A，如果A出错，则进行rollback回滚。

应该是最近同事写需求，修改了A，然后某种条件A报错了，然后操作的那张表发现所有的数据都没了。

来了解一下truncate的操作。

TRUNCATE

truncate是对数据进行删除的操作，可能会想到delete以及drop，通过比较三者来说说他们的区别。

delete语句执行删除是每次从表中删除一行，并且将此次删除操作作为事物记录在日志中保存以便进行回滚操作。

truncate则是一次性的删除表中所有的数据，而且不会将操作记录计入日志保存，通过再insert数据比较与delete的区别就是：delete之后再插入数据，自增的id是以删除之前的id开始增长的，但是truncate之后，是从0开始的。当表被TRUNCATE 后，这个表和索引所占用的空间会恢复到初始大小。

truncate会删除表中所有记录，并且将重新设置高水线和所有的索引，缺省情况下将空间释放到minextents个extent，除非使用reuse storage，。不会记录日志，所以执行速度很快，但不能通过rollback撤消操作（如果一不小心把一个表truncate掉，也是可以恢复的，只是不能通过rollback来恢复）。

所以对于存在外键约束的表，不能使用truncate，而应该使用不带where的delete。由于 TRUNCATE TABLE 不记录在日志中，所以它不能激活触发器。

drop语句将表所占用的空间全释放掉。包括这个表。

python中的四舍五入

发表于 2019-08-05 更新于 2019-08-14 分类于 Python

前言

最近工作过程中遇到了需要计算金钱的问题，由于有零有整，多次计算之后进行保留两位小数的时候，会发生计算不太准确的问题。
比如：
572.725需要保留两位小数，按照我们生活中来说，应该是572.73，但是在程序中最后保留的结果为572.72。

通过查资料，多数编程语言中，采用的不是我们生活中常用“四舍五入”，而是让统计数据更公平，降低舍入的误差的“奇进偶舍”的取舍方式，又称为“四舍六入五成双”。

奇进偶舍（四舍六入五成双）

“奇进偶舍”是一种比较精确比较科学的计数保留法，是一种数字修约规则。从另一种说法“四舍六入五成双”来看，就是遇到六才进一位，而遇到五需要进行别的操作。

举个例子🌰：

四舍六入五成双，高位为单数则进1凑成双数，高位为双数则不进位。

至于为什么要这么算，看一个百度百科说的例子：

1.15+1.25+1.35+1.45=5.2，若按四舍五入取一位小数计算：

1.2+1.3+1.4+1.5=5.4

按“四舍六入五成双”计算，1.2+1.2+1.4+1.4=5.2，舍入后的结果更能反映实际结果。

至于更深的原因，也没有去详细了解了，有兴趣的可以自己去查查看。下面来说说如果遇到了需要四舍五入的情况，需要怎么办。

Decimal

工作中总会遇到需要进行浮点数计算的时候，但是我们直接写在程序里面的浮点数，计算机是无法精确的使用二进制小数来进行表示。可以看一篇大佬的文章浮点数为什么不精确？，受益良多。

不知道在别的编程语言怎么处理，在python中通常是使用标准库Decimal来处理的。

需要进行小数位的保留的时候，使用quantize()方法。

quantize()提供了一个rounding参数rounding来决定在保留两位小数的时候进行的操作。

所以在使用的时候可以根据自己的需求来选择rounding的值。

什么是TOX

简单来说，TOX是一个虚拟环境管理和命令行测试工具，目标是为了自动化，标准化的进行Python程序的测试。作用如下：

  • 使用不同的Python版本和解释器检查项目的包安装是否正确。

  • 在每个环境中执行项目的测试工具，比如pytest等。

  • 处于CI（持续集成）之前，能大幅减少测试工具所需要的时间。
